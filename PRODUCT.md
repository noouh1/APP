# Product

## Register

product

## Users

People using the app to log in quickly, configure a destination site, and send that site to an in-app browser on iOS or Android. The primary workflow is short and task-oriented: authenticate, set a URL, choose a printer/device target, and open the configured web page.

## Product Purpose

Provide a simple mobile app with three clear screens: a social login entry point, a settings screen for the target web URL and device selection, and a web view screen that loads the saved site. Success means the flow feels immediate, stable, and easy to revisit without reconfiguration.

## Brand Personality

Practical, confident, familiar.

The interface should feel like a reliable tool rather than a novelty app. It should reduce friction, explain itself clearly, and keep the user moving through the setup and viewing flow without unnecessary decoration.

## Anti-references

Avoid flashy consumer-style login screens, decorative dashboard chrome, or overly experimental mobile navigation. Avoid anything that makes the app feel like a demo rather than a working utility.

## Design Principles

- Keep the main path obvious: login, configure, view.
- Prefer standard mobile controls and familiar interaction patterns.
- Surface loading, empty, and error states instead of hiding them.
- Make persisted settings easy to edit and safe to recover.
- Favor clarity and speed over visual surprise.

## Accessibility & Inclusion

Aim for WCAG 2.1 AA contrast, large touch targets, clear focus states, and readable copy. Support reduced motion and provide meaningful fallback states when device discovery or network access is unavailable.
